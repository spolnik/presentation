Intro

Old way

Problems

  • Hardcoded host within LTMs
  • Network changes required
  • Opening many firewall rules
  • Adding / Removing node is manual

New way

Benefits

No more network dependencies

  • No more of LTMs
  • Services are discovered
  • No more hardcoded hosts

Dynamic load balancing

You add or remove nodes and routing & load balancing apply it in real time

Smart healtchecks

Monitor your infra in real time, choose what you want to act on

Prod like infra everywhere

Keep same load balancing, routes and configuration across all environments

Hosted within Neo

No effort required, the only bits are to make it smarter

Adding new Service

Requirements

  • HTTPS + re-use app port
  • Cluster aware Consul agent on same machine
